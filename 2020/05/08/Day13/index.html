Day13 - 缓冲流，转换流，标准输入输出流，数据流，对象流，随机存取流

Posted on 2020-05-08 In Java SE

缓冲流

FileInputStream, FileOutputStream, FileReader, FileWriter 这些都是计算机与硬盘之间发生的Io操作。基于硬盘的读写相对比较慢，这个操作的速度受到硬盘的读写速度的制约，为了能够提高读写速度，从一定程度上绕过硬盘的限制，java提供一种缓冲流来实现。

缓冲流就是先把数据缓冲到内存里，在内存中去做io操作，基于内存的io操作大概能比基于硬盘的io操作快75000多倍。

缓冲字节流
缓冲字符流

缓冲流是把数据缓冲到内存中，使用flush()将会使内存中的数据立刻写出。

转换流

所有的文件都是有编码格式的，txt和java文件一般来讲有三种编码：

ISO8850-1，西欧编码，是纯粹英文编码，不适应汉字

GBK和UTF-8，这两种编码适用于中文和英文，我们一般使用UTF-8

转换流提供了在字符流和字节流之间的转换。当字节流中的数据都是字符的时候，使用转换流转换成字符流处理效率更高。

对象流

保存对象到硬盘（对象的持久化）和对象的网络传输，就产生了对象的输入与输出流。

序列化与反序列化都针对的是对象的各种属性，不包括类的属性。

如果需要让某个对象支持序列化机制，则必须让其类是可序列化的，为了让某个类是可序列化的，该类必须实现如下两个接口之一：

Serializable

Externalizable

注意：对象的序列化与反序列化使用的类要严格一致，包名，类型，类结构等等都要一致。

随机存取流
